# Story 1.2: User Registration as Employer/Agency

**Status:** in-progress

**Epic:** User Authentication & Onboarding (Epic 1)
**Priority:** P0 — Authentication is the foundation for all platform features
**Complexity:** Medium

---

## Story Requirements

### User Story

**As an** employer or agency,
**I want** to register my company so that I can post jobs and find workers.

### Acceptance Criteria

1. **Registration Form Components**
   - Form displays all required fields: full name, mobile number, email, password, location (district/city), company/agency name, industry type
   - Form includes role selection (pre-selected as "Employer/Agency" but visible)
   - Form has field validation with real-time feedback
   - Form submits only when all required fields are valid

2. **Input Validation**
   - All fields are required except industry type (optional)
   - Email must be unique and valid format
   - Password must meet minimum security requirements (min 8 chars, 1 uppercase, 1 number)
   - Mobile number must be valid Sri Lankan format
   - Company/agency name must be unique

3. **Email Verification**
   - After registration, user receives verification email
   - Account is inactive until email is verified

4. **Error Handling**
   - Error messages shown for invalid or missing fields
   - Duplicate email or company name triggers error

5. **Post-Registration Flow**
   - Successful registration redirects to company profile setup
   - User is prompted to complete company profile

6. **Security & Compliance**
   - Passwords are stored hashed (bcrypt)
   - Data privacy and GDPR compliance

## Dev Notes

### Architecture & Technology
- Frontend: React.js (TypeScript) with Material-UI components
- Backend: Node.js + Express.js (TypeScript)
- Database: PostgreSQL with SQL migrations
- Password hashing: bcryptjs (10 rounds)
- Email verification: Token-based (24hr expiry)
- Authentication: JWT with 24hr expiry

### Related Story Context
- Story 1.1 implements job seeker registration - reuse validation patterns
- Existing RegistrationForm.tsx handles job seeker flow - extend for employer
- Backend auth service and controller already handle user creation
- Email verification flow proven in story 1.1

### Key Technical Decisions
1. **Company table**: Create separate PostgreSQL table for company data (normalized design)
2. **Role field**: Use existing 'role' in users table (already supports 'employer')
3. **Industry type**: Dropdown with predefined options (better data quality than free text)
4. **Uniqueness validation**: Company name unique at registration time via DB constraint
5. **Form flow**: Conditional rendering based on selected role

### Database Changes Required
- Create new migration: `003-create-companies-table.sql`
- Add unique constraint on company name
- Foreign key relationship: companies.user_id → users.id

### Frontend Changes Required
- Extend RegistrationFormData type to include employer fields
- Add EmployerRegistrationForm or conditional fields in RegistrationForm
- Add industry type constants and dropdown
- Update registrationAPI.ts to handle employer data

### Backend Changes Required
- Extend registration controller to handle company creation
- Add createCompany function in authService.ts
- Add validation for company name uniqueness
- Extend RegistrationRequest type to support company fields
