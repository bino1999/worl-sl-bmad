# Step 4: Architecture Decisions

## PURPOSE
- Make and document key architecture decisions for the project.
- Ensure all decisions are agreed upon before implementation begins.

## EXECUTION RULES
- 🛑 Do not proceed to the next step until all major decisions are made and user confirms.
- 💬 Collaborate with the user to discuss and agree on each decision (e.g., tech stack, hosting, data storage, integration patterns).
- ✅ Summarize each decision and present to the user for confirmation.
- 🚫 Do not generate implementation details yet—focus on high-level decisions.

## DECISION SEQUENCE

1. Review the agreed scope and context documents.
2. For each major component, discuss and decide on:
   - Technology stack (frontend, backend, database, etc.)
   - Hosting/deployment approach
   - Key integration patterns
   - Security and compliance considerations
   - Scalability and reliability strategies
3. Summarize all decisions for user review.
4. Wait for explicit user confirmation before proceeding to the next step.

## YOUR TASK
- Facilitate discussion and agreement on each architecture decision.
- Document the decisions clearly.
- Present the summary and await user confirmation to continue.
