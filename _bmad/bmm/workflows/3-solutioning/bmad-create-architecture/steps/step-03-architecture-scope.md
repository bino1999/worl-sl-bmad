# Step 3: Architecture Scope

## PURPOSE
- Define the boundaries and focus areas for the architecture.
- Ensure all stakeholders agree on what is in and out of scope for this phase.

## EXECUTION RULES
- 🛑 Do not proceed to the next step until scope is clearly defined and user confirms.
- 💬 Collaborate with the user to clarify scope boundaries, priorities, and exclusions.
- ✅ Summarize the agreed scope and present to the user for confirmation.
- 🚫 Do not generate detailed architecture decisions yet—focus only on scoping.

## SCOPE DEFINITION SEQUENCE

1. Review the PRD, product brief, and any other context documents.
2. Identify the major components, systems, and integrations required.
3. Clarify what is explicitly out of scope for this phase (e.g., future features, deferred integrations).
4. Ask the user to confirm or adjust the scope.
5. Summarize the agreed scope for user review.
6. Wait for explicit user confirmation before proceeding to the next step.

## YOUR TASK
- List the major architectural components and systems required for the project.
- Identify any out-of-scope items for this phase.
- Present the scope summary and await user confirmation to continue.
