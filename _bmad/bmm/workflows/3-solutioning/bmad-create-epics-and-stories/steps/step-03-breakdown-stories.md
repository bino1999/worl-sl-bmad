# Step 3: Break Down Epics into User Stories

## STEP GOAL:
Transform each approved epic into a set of actionable user stories, each with clear acceptance criteria. Stories should be small, testable, and deliver user or system value.

## EXECUTION RULES:
- 📖 Read this step file fully before taking any action
- 🛑 Do NOT skip or combine epics—address each epic in sequence
- 💬 Collaborate with the user to validate and refine stories and acceptance criteria
- 📝 For each story, provide:
   - Story Title
   - User Story (As a [user], I want [goal], so that [value])
   - Acceptance Criteria (bullet list, clear and testable)
   - Link to parent epic
- 💾 Append all stories to {planning_artifacts}/epics.md under a new section, and update frontmatter stepsCompleted
- ⏸️ When finished, present a menu: [C] Continue to next step | [E] Edit stories | [R] Review epics again
- 🚫 Do NOT proceed to the next step until user selects 'C'

## STORY BREAKDOWN PROCESS:
1. Review each epic in the current epics.md
2. For each epic, propose a set of user stories with acceptance criteria
3. Present all proposed stories to the user for feedback and adjustment
4. Only after user approval, append the finalized stories section to epics.md and update stepsCompleted

---

*When ready, type 'C' to continue, 'E' to edit stories, or 'R' to review epics again.*
