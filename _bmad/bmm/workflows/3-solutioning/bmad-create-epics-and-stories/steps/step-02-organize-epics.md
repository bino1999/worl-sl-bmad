# Step 2: Organize Requirements into Epics

## STEP GOAL:
Transform the extracted requirements into a clear, actionable set of epics. Each epic should represent a major user or system value stream and group related requirements logically.

## EXECUTION RULES:
- 📖 Read this step file fully before taking any action
- 🛑 Do NOT create user stories yet—focus only on epic-level grouping
- 💬 Collaborate with the user to validate and refine epic groupings
- 📝 Document each epic with a title, description, and the requirements it covers
- 💾 Update {planning_artifacts}/epics.md by appending the organized epics section and update frontmatter stepsCompleted
- ⏸️ When finished, present a menu: [C] Continue to next step | [E] Edit epics | [R] Review requirements again
- 🚫 Do NOT proceed to the next step until user selects 'C'

## EPIC ORGANIZATION PROCESS:
1. Review all extracted requirements in the current epics.md
2. Propose initial epic groupings (e.g., Authentication, Job Posting, Search & Discovery, Application Flow, Messaging, Reviews, Notifications, Admin, etc.)
3. For each epic, provide:
   - Epic Title
   - Short Description
   - List of covered requirements (by reference or summary)
4. Present proposed epics to the user for feedback and adjustment
5. Only after user approval, append the finalized epics section to epics.md and update stepsCompleted

*When ready, type 'C' to continue, 'E' to edit epics, or 'R' to review requirements again.*
